# BLOCKING CONSTRAINTS - Read Before Anything Else

> These are not suggestions. Each was discovered/reasoned through this session.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: isSyncTrusted-may-fail-closed-on-schedule - Do NOT wire `isSyncTrusted(process.env)` into `cleanup/index.ts run()` until a test PROVES it returns `{trusted:true}` on a real `schedule`-event payload. On `schedule`, isSyncTrusted (sync-gate.ts) requires `repository.default_branch` read from `GITHUB_EVENT_PATH` AND `GITHUB_REF_NAME===default_branch` AND `GITHUB_REF` startsWith `refs/heads/`. It is UNVERIFIED that the schedule-event payload reliably carries `repository.default_branch`. If it does not, the gate returns `not-default-branch`, cleanup SILENTLY no-ops every run, and the mirror leaks toward the 1000-asset cap - violating the retention-locked requirement (the exact failure retention.ts guards). Structural mitigation: write the schedule-event proof test FIRST; if isSyncTrusted is unsafe there, use a narrower schedule-only assertion that cannot fail-closed on the happy path.

**Do not proceed until the box is checked.**
