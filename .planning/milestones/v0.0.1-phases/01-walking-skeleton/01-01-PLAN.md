## Phase Goal

**As a** developer or CI job running Nx against a self-hosted remote cache, **I want to** run a
loopback HTTP server that speaks the Nx `GET`/`PUT /v1/cache/{hash}` contract end-to-end against
a trivial in-process store, **so that** the protocol (status codes, auth, hardening) is proven
and conformance-locked before any real storage backend is built.

> Slice for this plan: the vessel. This is the Wave-1 scaffold that creates the `nx g @nx/js:lib`
> library the whole walking skeleton lives in and proves the Vitest harness runs green, so every
> subsequent first-written (RED) test has a place to land.

`nx g @nx/js:lib`, let `@nx/js/typescript` + `@nx/vitest` infer the `build`/`typecheck`/`test`
targets (D-02, no hand-authored `project.json`), confirm the generator wired it into the root
TS-solution `references[]`, and confirm `npx nx test github-cache` runs green.

Purpose: SC1's "a fresh library (created via `nx g @nx/js:lib`)" clause and the vessel SRV-01's
loopback server is built into. It also creates the Vitest infrastructure that VALIDATION.md
marks as Wave 0 for every SRV-01..05 + TEST-07 first-written test. Phase 1 introduces ZERO new
dependencies (D-01, D-03): `node:http`/`node:crypto` are stdlib and vitest/swc/typescript are
already in the shell devDependencies.

Output: a green, inferred-target `@op-nx/github-cache` project ready to receive the TDD source
modules in waves 2-4.
