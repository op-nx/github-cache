<objective>
Deliver the sync/publish gate as a pure, injectable predicate `isSyncTrusted(env, readDefaultBranch?)` that is a SEPARATE trust boundary from the write gate (D-01 / TRUST-02): literally `{push, schedule}` inside GitHub Actions AND the current ref is the repository default branch. This is the first statement of the publish path; everything downstream (04-04 engine, 04-06 CI job) gates on it.

Purpose: Phase 5 widens the WRITE allowlist to pull_request/release. A shared predicate would silently widen SYNC at the same time, recreating the exact CREEP precondition ADR control C2 exists to prevent. Keeping the sync gate a distinct source of truth is load-bearing.
Output: `lib/sync-gate.ts` + `lib/sync-gate.spec.ts`.
</objective>

## Phase Goal

**As a** repo maintainer running Nx builds in CI, **I want to** mirror CI-produced cache entries to GitHub Releases on default-branch pushes and prune them safely by age, with loud failures and graceful cap handling, **so that** cross-context readers get correct cross-OS hits and the cache never breaks a build or deletes live data.

This plan delivers the first vertical slice: the gate that decides whether a run is publish-eligible at all.
