## Phase Goal

**As a** repository maintainer adopting @op-nx/github-cache, **I want to** have write-trust widened to `pull_request`/`release` only where GitHub's default-branch cache guard exists (fail-closed on GHES), backed by a single-source allowlist, a server-produced-key mirror filter, and an installable PPE-hygiene gate, **so that** contributor PR/release runs can write cache without opening a CREEP hole and only genuine cache keys are ever mirrored.

<objective>
Make the widened `trust.ts` allowlist the single source of truth (D-05) and generate a committed, dependency-free CommonJS copy of the allowlist + host gate from it (D-06), guarded by a `selfcheck.cjs` byte-drift tripwire wired into CI (D-07) and a Vitest semantic-parity spec (the load-bearing behavioral guard). This rebuilds the PoC's `index.cjs` + `selfcheck.cjs` pattern that Phase 0 tore down, eliminating the dual-`TRUSTED_EVENTS`-copy drift hazard the ADR named as a root-cause fix.

Purpose: TRUST-04 - the pre-`npm ci` dependency-free action copy must never silently diverge from the server's write gate. The copy exists because JS actions run before `npm ci`.
Output: selfcheck.cjs (generator + drift diff), committed trust.generated.cjs, trust.generated.spec.ts (semantic parity), ci.yml + package.json + .fallowrc.jsonc wiring.
</objective>
