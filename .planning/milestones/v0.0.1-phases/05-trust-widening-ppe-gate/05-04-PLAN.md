## Phase Goal

**As a** repository maintainer adopting @op-nx/github-cache, **I want to** have write-trust widened to `pull_request`/`release` only where GitHub's default-branch cache guard exists (fail-closed on GHES), backed by a single-source allowlist, a server-produced-key mirror filter, and an installable PPE-hygiene gate, **so that** contributor PR/release runs can write cache without opening a CREEP hole and only genuine cache keys are ever mirrored.

<objective>
Ship the adopter-facing PPE-hygiene gate as a COMPOSITE ACTION (`ppe/action.yml`) that self-installs exact-pinned `zizmor` (1.27.0) + `actionlint` (1.7.12) and runs them advisory for the named unsafe-trigger patterns (no `pull_request_target`+PR-checkout; no `issue_comment`/`workflow_run` executing PR code). Guard its structure with a config-assertion spec (the local Wave 0 gate) and prove it runs against a known-unsafe fixture via an advisory CI job (the first-push closing proof).

Purpose: TRUST-06 / ADR C4 - a shipped, installable best-effort/advisory defense-in-depth gate adopters fold into their own security job. It is explicitly NOT the containment control (that stays TRUST-02 + default-branch protection).
Output: ppe/action.yml (composite), ppe/fixtures/unsafe-workflow.yml, ppe-action.spec.ts (config-assertion), an advisory ci.yml PPE job.
</objective>
