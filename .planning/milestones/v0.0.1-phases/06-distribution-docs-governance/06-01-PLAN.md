<objective>
MVP user story: As an external repo maintainer, I want to consume @op-nx/github-cache as a
publish-ready npm package and a `uses:`-consumable JS Action, so that I can turn on
GitHub-backed Nx remote caching from another repo without building this workspace first.

This plan ships the two distribution channels from one codebase (per 06-RESEARCH "System
Architecture"): (1) the npm tarball (`files:["dist"]` + `bin`), and (2) a NEW `uses:`-consumable
JS action at repo-root `start-cache-server/` whose committed, esbuild-bundled entry is kept
honest by a CI drift guard. It flips the package from `private:true` to publish-READY (D-13) and
bundles the MIT LICENSE into the tarball (GOV-02).

Purpose: `dist/` is gitignored (`.gitignore:4`), yet a `uses:` action resolves `main:` from the
git ref -- never from npm and never after a build. The internal dogfood action
(packages/github-cache/action.yml) works only because this repo builds first; an external
consumer cannot. So the consumer action MUST ship a committed, dependency-inlined single file
(D-09). The npm tarball is a separate channel governed by `files`, not `.gitignore`.

Output: publish-ready packages/github-cache/package.json, the committed consumer action
(start-cache-server/*), the bundle build+drift scripts, the esbuild pin guard, the npm-pack
file-list guard, and the CI wiring (drift job + a background-step live smoke).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo -> published npm tarball | Files leave the repo into a public tarball adopters install; anything not excluded by `files` leaks |
| repo git ref -> consumer `uses:` action | External repos resolve start-cache-server/index.js from the ref; a stale/tampered bundle runs in their CI |
| workspace build tooling -> committed bundle | The esbuild devDependency + its output become a supply-chain input to every consumer |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-01-01 | Information Disclosure | npm tarball (files list) | high | mitigate | `files: ["dist"]` allow-list + pack-check.cjs asserts src/.github/.planning/nx.json/.env excluded, dist/LICENSE/README included (dogfood-stays-local) |
| T-06-01-02 | Tampering (supply chain) | esbuild devDependency | high | mitigate | esbuild exact-pinned (OK legitimacy verdict) + pinned-deps.spec.ts fails on a range; postinstall is the benign native-binary fetch, devDependency only, never shipped |
| T-06-01-03 | Tampering | committed start-cache-server/index.js | high | mitigate | `check:action` CI job rebuilds with esbuild and `git diff --exit-code` -- a hand-edited or stale bundle fails CI (selfcheck.cjs pattern) |
| T-06-01-04 | Information Disclosure | package.json author / LICENSE holder | high | mitigate | author is the approved public gmail ONLY; no work email/domain; the 06-03 email allowlist-inversion guard backstops this repo-wide |
| T-06-01-05 | Elevation of Privilege | background-step smoke job permissions | low | accept | Smoke uses the workflow-default contents:read + inherited runtime; no fork-PR token, push-gated; rationale: loopback-only, per-process CSPRNG token, no write scope requested |
</threat_model>
