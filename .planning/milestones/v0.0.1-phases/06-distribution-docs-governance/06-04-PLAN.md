and a copyable minimal example, so that I can turn on the default CI cache fast and grow into the
opt-in store/sync/cleanup features without reverse-engineering this repo's dogfood config.

guide (opt-in RO store/sync/cleanup + the `&` fallback + JS-action rationale, D-03/D-09), the
config reference (every resolve* knob + Nx client vars + the two required notes, D-07/DOCS-02),
and a minimal example adopter config distinct from the dogfood config (DOCS-04). It documents the
DOCS-06 background-step consumption pattern (the JS action itself ships in 06-01).

Purpose: the default CI-RW quickstart is the primary product (D-02); advanced features are opt-in.
Docs render the SETTLED model -- they describe, they do not change behavior.

Output: README.md, docs/advanced.md, docs/configuration.md, docs/examples/*, and a docs-adoption
content guard spec.
</objective>
