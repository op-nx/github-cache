## Phase Goal

**As a** developer or CI job running Nx against a self-hosted remote cache, **I want to** run a
loopback HTTP server that speaks the Nx `GET`/`PUT /v1/cache/{hash}` contract end-to-end against
a trivial in-process store, **so that** the protocol (status codes, auth, hardening) is proven
and conformance-locked before any real storage backend is built.

> Slice for this plan: the spine. After this plan a developer CAN do a real, authenticated cache
> round-trip: `PUT /v1/cache/{hash}` stores bytes and returns `200`; `GET /v1/cache/{hash}`
> returns them with `Content-Length`; unauthenticated requests get `401`. This is the thinnest
> viable end-to-end slice (MVP task order: failing E2E test -> thinnest slice that passes).

<objective>
Build the walking skeleton's end-to-end happy path test-first (TDD): the `CacheBackend` port
(D-03), a trivial writable `Map`-backed adapter, and a `node:http` server (D-01, zero deps) that
authenticates a bearer token in constant time (SRV-02), binds loopback only (SRV-01), and
round-trips `PUT`->`200` / `GET`->`200`+`Content-Length` against the backend injected at
construction (the D-04 seam — RW/RO is which factory the caller passes, never a request flag).

Purpose: SC2's authenticated `200` round-trip + `404`-on-miss + `401`-on-unauth, and SC1's
SRV-01/SRV-02 hardening — each covered by a first-written test.

Output: `types.ts`, `memory-backend.ts` (writable form), `server.ts` (happy path + auth gate),
and their RED specs, with `npx nx test github-cache` green.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Nx client -> server | The `Authorization: Bearer` header and the request line cross here from an untrusted caller into the server process. |
| server process -> loopback socket | The server exposes a listening socket; it must be reachable on loopback only, never a routable interface. |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |

| T-1-01 | Spoofing / Elevation of Privilege | `createCacheServer` listen bind (SRV-01) | high | mitigate | Bind `127.0.0.1` only; never `0.0.0.0`. First-written SRV-01 test asserts `server.address().address === '127.0.0.1'`. |
