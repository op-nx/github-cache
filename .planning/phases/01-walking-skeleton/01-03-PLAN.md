## Phase Goal

**As a** developer or CI job running Nx against a self-hosted remote cache, **I want to** run a
loopback HTTP server that speaks the Nx `GET`/`PUT /v1/cache/{hash}` contract end-to-end against
a trivial in-process store, **so that** the protocol (status codes, auth, hardening) is proven
and conformance-locked before any real storage backend is built.

> Slice for this plan: hardening. This refines the Plan 02 round-trip with the Core-Value safety
> behaviors — reject malformed input (400), cap oversized bodies (413), degrade read faults to a
> MISS (404) while writes fail closed, and complete the status contract (409 no-override, 403
> read-only). MVP task order step 4: validation + error states on the existing slice.

<objective>
Harden the Plan 02 server test-first (TDD) into the full Nx status contract: validate the
`{hash}` path (bounded hex) and reject malformed input with `400` BEFORE any backend call
(SRV-03); cap request bodies at `MAX_CACHE_BODY_BYTES` (2 GiB) with a `Content-Length` precheck +
streaming socket-destroy at overflow so a body is never buffered unbounded (SRV-04); degrade any
`backend.get` fault to a `404` MISS while letting `backend.put` faults propagate (best-effort
read, fail-closed write — SRV-05, D-06); and complete the `PutResult`->status map with a
`never`-typed exhaustiveness guard (D-06) so `'conflict'`->`409` and `'forbidden'`->`403`, the
`403` fed by a read-only backend factory (the D-04 seam, no caller-facing mode flag).

Purpose: SRV-03, SRV-04, SRV-05 and the `409`/`403` half of SC2 — each covered by a first-written
test.

Output: the completed `server.ts` pipeline, `createReadOnlyMemoryBackend()`, and their RED specs,
with `npx nx test github-cache` and `npx nx typecheck github-cache` green.
</objective>
