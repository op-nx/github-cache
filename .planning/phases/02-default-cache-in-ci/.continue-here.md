> Discovered through actual failure this session. Acknowledge before proceeding.

- [ ] CONSTRAINT: **Cross-OS lockfile -- never `npm install` on Windows to change `package-lock.json`.** A Windows workspace-scoped install prunes the `@emnapi` WASM-fallback optional deps (`@oxc-resolver`/`@rolldown` `*-wasm32-wasi` subtrees) that Linux `npm ci` requires -- it turned every CI job red. STRUCTURAL MITIGATION: if any dependency changes, regenerate the lockfile in a Linux `node:24` Docker container with `node_modules` masked (`-v /app/node_modules`, `npm install --package-lock-only`), then prove it with `npm ci` (which never rewrites the lockfile) on BOTH Linux (Docker) and Windows (local) before committing. The post-completion audits below do NOT change deps, so they will not trip this -- but do not "tidy" the lockfile.

**Do not proceed until the box is checked.**
