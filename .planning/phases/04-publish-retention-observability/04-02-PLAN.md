## Phase Goal

**As a** repo maintainer running Nx builds in CI, **I want to** mirror CI-produced cache entries to GitHub Releases on default-branch pushes and prune them safely by age, with loud failures and graceful cap handling, **so that** cross-context readers get correct cross-OS hits and the cache never breaks a build or deletes live data.

This slice makes the reader resolve HITs across the whole retention window, not just the current month.

## Artifacts this plan produces

- `resolveMaxAgeDays(env)` — one-knob resolver, default 30, clamp 365, reject NaN/<=0 (new symbol)
- `shardTagsForWindow(maxAgeDays, now?)` — newest-first calendar-month shard tags (new symbol)
- `shardTag(date?)` — current-month tag, MOVED from releases-backend.ts to retention.ts (same behavior)
- `CACHE_MIRROR_MAX_AGE_DAYS` — the single coupled retention env knob
- `packages/github-cache/src/lib/retention.ts`, `retention.spec.ts`; modified `releases-backend.ts` + spec
