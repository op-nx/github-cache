<objective>
Deliver the age-based cleanup engine `cleanupMirror(client, maxAgeDays)` behind a narrow injected client (D-06/D-09/D-10): a fail-loud LIST phase that materializes every `cache-mirror-*` release + asset before any deletion, and a per-item-isolated DELETE phase that prunes assets older than the window and exits non-zero on aggregate failure. Plus the shared `octokitFault` test factory used here and by the publish engine (04-04).

Output: `cleanup/cleanup.ts` + spec; `test/octokit-fault.ts`.
</objective>

## Phase Goal

**As a** repo maintainer running Nx builds in CI, **I want to** mirror CI-produced cache entries to GitHub Releases on default-branch pushes and prune them safely by age, with loud failures and graceful cap handling, **so that** cross-context readers get correct cross-OS hits and the cache never breaks a build or deletes live data.

This slice delivers the prune logic that keeps storage bounded without ever deleting live data on a partial listing.
