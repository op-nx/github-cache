## Phase Goal

**As a** repo maintainer running Nx builds in CI, **I want to** mirror CI-produced cache entries to GitHub Releases on default-branch pushes and prune them safely by age, with loud failures and graceful cap handling, **so that** cross-context readers get correct cross-OS hits and the cache never breaks a build or deletes live data.

This slice is the mirror engine: it turns restored cache bytes into first-write-wins Release assets, safe at the storage caps.

## Artifacts this plan produces

- `publishMirror(client, options)` — enumerate -> filter (nx-cache-) -> restore (this OS) -> cap/size guard -> first-write-wins upload; returns mirrored/skipped counts (new symbol)
- `PublishClient` — narrow injected client interface (listCacheEntries / getReleaseByTag / createRelease / listReleaseAssets / uploadReleaseAsset) (new symbol)
- `RELEASE_ASSET_MAX_BYTES` — the ~2 GiB pre-upload boundary constant (new symbol)
- `packages/github-cache/src/publish/publish-mirror.ts`, `publish-mirror.spec.ts`

| Boundary | Description |
|----------|-------------|
| Actions-cache entries -> public Release assets | Only server-produced (nx-cache-) keys may be mirrored to world-readable assets (public repo) |
| upload host -> stored asset | A ~2 GiB boundary artifact must fail loud, never truncate/drop |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-11 | Information Disclosure | mirroring unrelated CI keys | high | mitigate | D-16 nx-cache- prefix filter (the cheap TRUST-08 subset); only server-produced keys become public assets. Full single-source filter + parity is Phase 5/TRUST-08 (C16); safe here because the repo is PUBLIC |
| T-04-12 | Tampering | overwrite of an existing asset | high | mitigate | First-write-wins via a pre-list existence check; a duplicate-upload 422 already_exists is a benign no-op, discriminated structurally (TRUST-07/D-05) |
| T-04-13 | Tampering | real fault treated as absence/already-exists | high | mitigate | statusOf duck-type on error.status: only 404 is absence; a 422 is benign only when already_exists / pre-listed; every other status is surfaced (ROBUST-01, Pitfall 1) |
| T-04-14 | Denial of Service | silent truncation at ~2 GiB / hard-fail at 1000-cap | medium | mitigate | Deterministic pre-upload byte-length fail-loud (never truncate); 1000-cap degrades to skip-and-warn (never breaks the build) — D-12/D-11 |
