## Nyquist Classification

- **Fault-injection (highest priority):** RETAIN-01/TEST-04 (mid-pagination abort -> zero deletions — the ROADMAP's named cleanup risk); ROBUST-01/TRUST-07 (404 vs 422 vs 5xx branches); ROBUST-02 (~2 GiB boundary). Drive by making the injected fake throw `{ status, response: { data: { errors: [{ code: 'already_exists' }] } } }`.
- **Property / boundary:** `shardTagsForWindow` across a Dec->Jan boundary, a 28-day February, and exactly-30-day windows (assert newest-first, no under-scan); `resolveMaxAgeDays` NaN/negative/over-ceiling -> default/clamp; the 1000-asset cap at 999/1000/1001.
- **Ordinary unit (injected client):** TEST-03 happy-path publish; TEST-06 age prune + within-window retain + `put()===403`; TRUST-02 event/ref matrix; OBS-01 annotation/summary emission.
- **Live / integration (CI, not local Vitest):** the cross-OS round-trip through the real Releases mirror — model as a CI job pair like `dogfood-seed`/`dogfood-verify`.

## Audit Trail (retroactive, post-execution -- gsd-nyquist-auditor, 2026-07-20)

**Method:** Independently re-derived coverage for all 11 Phase 4 requirement IDs (TEST-03,
TEST-04, TEST-06, ROBUST-01, ROBUST-02, ROBUST-05, TRUST-02, TRUST-07, RETAIN-01, RETAIN-03,
OBS-01) by reading every production source file this phase touched
(`sync-gate.ts`, `retention.ts`, `cleanup.ts`, `publish-mirror.ts`, `cleanup/index.ts`,
`action/index.ts`, `cleanup.yml`) alongside every corresponding `*.spec.ts`, all six
`04-0N-PLAN.md`/`SUMMARY.md` pairs, `04-VERIFICATION.md`, and `REQUIREMENTS.md` -- not accepting
the orchestrator's preliminary gap read at face value. Ran `npx nx test github-cache` fresh
(`--skip-nx-cache`), `npx nx typecheck github-cache --skip-nx-cache`, `npx nx build github-cache`,
and `npm run fallow:ci` to independently confirm green before and after adding new coverage.

**Findings, per requirement:**

- **TEST-03, ROBUST-01, ROBUST-02, ROBUST-05, TRUST-02, TRUST-07, RETAIN-01, TEST-06, OBS-01** --
  CONFIRMED covered by genuinely adversarial, non-vacuous unit tests (many spec comments
  explicitly document the non-vacuous negative case, e.g. `releases-backend.spec.ts`'s
  cross-OS MISS test calling out a prior tautological-guard trap). No gap; no new test needed.
- **TEST-04** -- CONFIRMED covered at the `cleanupMirror` engine level (per-item isolation +
  aggregate `core.setFailed`), which is where this behavior actually lives; `cleanup/index.ts`
  (the "bin wrapper" named in REQUIREMENTS.md) is thin untested glue over the engine, already
  flagged as an INFO-severity (non-blocking) anti-pattern in `04-VERIFICATION.md`. Not
  re-litigated here: the requirement's observable behavior (isolation + non-zero exit) is
  proven; the adapter wiring itself is typecheck-verified only, consistent with the
  injected-client-seam design used throughout this phase.
- **RETAIN-03** -- GAP CONFIRMED and FILLED. Was manual-only (`cleanup.yml` review). Generated
  `src/cleanup/cleanup-workflow.spec.ts` (4 new tests, RED-then-GREEN verified against a
  scratchpad harness that mutated a copy of the config text -- never the real read-only
  `cleanup.yml` -- to prove each assertion fails under: `cancel-in-progress: true`, an added
  `actions: read` scope, a swapped-in PAT secret, a `push` trigger, and comment-only mentions of
  the same strings with the real directive absent). Converts this requirement from
  manual-review-only to automated (`npx nx test github-cache`).
- **cross-OS round-trip (deferred from P3)** -- CONFIRMED genuinely CI-only. Every sub-component
  (sync gate, both engines, the reader's cross-OS HIT/MISS branches, the retention window walk)
  is unit-proven with fault-shaped fakes; the actual live wire crossing requires a real
  default-branch push against live GitHub Releases + Actions-cache services on both runner OSes.
  Synthesizing a fake for this would produce a false COVERED (the orchestrator's gap note
  correctly flagged this risk). Left manual-only, matching `04-VERIFICATION.md`'s own
  `human_needed`/`PRESENT_BEHAVIOR_UNVERIFIED` classification for the identical item.

**Result:** 10/11 requirements were already automated; 1/11 (RETAIN-03) had a genuine automated-
coverage gap, now filled. 1 additional item (the live cross-OS round-trip) is not one of the 11
requirement IDs directly -- it is the deferred-from-Phase-3 round-trip referenced in the Wave 0
table -- and is correctly manual-only/CI-integration, not a local Vitest gap.
