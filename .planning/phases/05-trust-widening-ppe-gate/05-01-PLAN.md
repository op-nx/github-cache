## Phase Goal

**As a** repository maintainer adopting @op-nx/github-cache, **I want to** have write-trust widened to `pull_request`/`release` only where GitHub's default-branch cache guard exists (fail-closed on GHES), backed by a single-source allowlist, a server-produced-key mirror filter, and an installable PPE-hygiene gate, **so that** contributor PR/release runs can write cache without opening a CREEP hole and only genuine cache keys are ever mirrored.

> This plan is the FIRST slice (D-09): the server-produced-key filter must ship before any private-repo mirror is enabled, or an unrelated hex-keyed CI artifact would leak as a world-readable Release asset.

<objective>
Promote the `nx-cache-` prefix (authored today in exactly two production places: `actions-cache-backend.ts` and `publish-mirror.ts`) into ONE shared leaf module `src/lib/cache-key.ts` that owns the prefix, the key builder, the shared `HASH_PATTERN`, and a `isServerProducedKey` filter that validates prefix + a genuine hash suffix. Refactor the backend, the publish path, and the server to consume the single source, so the mirror publishes ONLY server-produced keys and never "any 1-512 hex" foreign artifact.

Purpose: TRUST-08 / ADR C16 - close the info-disclosure gap where a foreign hex-keyed CI cache entry could be mirrored as a public Release asset. This is the D-09 sequence-first deliverable.
Output: `cache-key.ts` leaf + spec; refactored backend/publish/server consumers; hex-corrected publish fixtures.
</objective>
