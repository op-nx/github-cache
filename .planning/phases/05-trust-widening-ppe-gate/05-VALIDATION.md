## Audit Trail (retroactive, post-execution -- gsd-nyquist-auditor, 2026-07-20)

**Method:** Independently re-derived coverage for all 4 Phase 5 requirement IDs (TRUST-01,
TRUST-04, TRUST-06, TRUST-08) by reading every production source file this phase touched
(`trust.ts`, `cache-key.ts`, `select-backend.ts`, `selfcheck.cjs`, `trust.generated.cjs`,
`ppe/action.yml`) alongside every corresponding `*.spec.ts`, all four `05-0N-PLAN.md`/`SUMMARY.md`
pairs, `05-VERIFICATION.md`, `05-SECURITY.md`, and `REQUIREMENTS.md` -- not accepting the
orchestrator's preliminary gap read at face value. Ran `npx nx test github-cache --skip-nx-cache`
fresh (bypassing Nx's task cache to force a genuine re-execution, not a replayed cached result),
`node packages/github-cache/selfcheck.cjs`, `npx nx typecheck github-cache --skip-nx-cache`,
`npm run fallow:ci`, and `npx nx format:check --all` to independently confirm green before and
after adding new coverage.

**Findings, per requirement:**

- **TRUST-01** -- CONFIRMED covered by genuinely adversarial, non-vacuous unit tests: the host
  matrix in `trust.spec.ts` exercises every ON host (`github.com`, a real `*.ghe.com` subdomain,
  `http://github.com` to prove scheme-agnostic host matching) and every fail-closed case named in
  the plan (GHES, empty, malformed, unset, `github.com.attacker.com` substring-spoof,
  `notghe.com`/`ghe.com` bare-label spoof). The dangerous trio + unlisted events are tested WITH a
  `github.com` host present (non-vacuous: proves the host does not rescue them). The sync-gate
  cross-check imports the real `isSyncTrusted` and proves it still refuses `pull_request`/`release`
  on a github.com default-branch run (ADR C2 separation held). `select-backend.spec.ts` proves the
  widening flows end-to-end through `selectBackend` (writable on github.com, forbidden on GHES).
  No gap; no new test needed.
- **TRUST-08** -- CONFIRMED covered. `cache-key.spec.ts`'s admit/reject matrix includes the hard
  edge the D-08 hardening exists for (`nx-cache-h1`/`nx-cache-ABC` rejected -- not just
  `startsWith`), the round-trip property (`isServerProducedKey(cacheKeyFor(h))` for `'0'`,
  `'abc123'`, and a 512-char hash), and a STRICT cross-file single-source count (reads
  `cache-key.ts` + `actions-cache-backend.ts` + `publish-mirror.ts` from disk, comment-strips, and
  asserts the authored prefix literal totals exactly 1, located in `cache-key.ts`). No gap.
- **TRUST-06 (structure)** -- CONFIRMED covered. `ppe-action.spec.ts` reads the tracked
  `ppe/action.yml` from disk (comment-stripped, so the file's own `#` prose cannot make an
  assertion vacuous) and pins the composite form, both exact tool versions, the `--no-exit-codes`
  advisory switch, an advisory/not-containment marker, and the per-step `shell:`/no-top-level-`env:`
  structural requirement. 05-04-SUMMARY.md documents this was independently mutation-proven at
  execution time (a pin-version edit failed the spec after an Nx-cache false-pass was caught and
  corrected with `--skip-nx-cache`). No gap.
- **TRUST-06 (live behavior)** -- CONFIRMED genuinely live-CI-only, matching the orchestrator's own
  gap note. The composite action self-installs `zizmor`/`actionlint` via `pipx`/a downloaded
  install script and executes them against a fixture -- this cannot be exercised in local Vitest
  without either (a) actually installing these external tools into this audit's environment and
  simulating a GitHub Actions composite-action runtime (`inputs.*`, `RUNNER_TEMP`), which is a live
  integration test masquerading as a unit test, or (b) stubbing/mocking the tool invocations, which
  would prove nothing about whether the REAL tools actually flag the REAL fixture -- a false
  COVERED. `ci.yml`'s `ppe` job is verified wired (advisory, non-blocking, targets
  `ppe/fixtures/unsafe-workflow.yml` which is verified to sit outside `.github/workflows` so
  GitHub never executes it as a real workflow). Left `human_needed`/live-CI, matching
  `05-VERIFICATION.md` and 05-04-SUMMARY.md's own classification for the identical item -- not
  synthesized, not skipped-without-escalation.
- **TRUST-04** -- GAP CONFIRMED and FILLED. The semantic-parity leg (`trust.generated.spec.ts`,
  147 tests across the full `GITHUB_ACTIONS` x `GITHUB_EVENT_NAME` x `GITHUB_SERVER_URL` matrix,
  plus a non-vacuous both-verdicts-appear guard and deep-equal allowlist arrays) and the
  in-sync/exit-0 leg of `selfcheck.cjs` (verified live: `node packages/github-cache/selfcheck.cjs`
  -> exit 0) were already solidly covered. But the requirement's own must-have --
  "selfcheck.cjs ... exits 1 (with a stderr message) when the committed copy has drifted" -- had
  ONLY ever been proven by a one-time MANUAL mutation-and-revert, recorded as prose evidence in
  05-03-SUMMARY.md (`kind: manual, human_judgment: true` in that SUMMARY's coverage table), never by
  a repeatable automated test. This is the requirement's hard behavioral edge (the whole point of a
  drift tripwire is that it actually trips), and nothing in the existing 424-test suite would catch
  a future regression to the comparison logic itself (e.g. a refactor that silently always returns
  "in sync"). Generated `packages/github-cache/src/selfcheck.spec.ts` (4 new tests): drives the REAL
  `selfcheck.cjs` as a child process against an ISOLATED TEMP COPY of the three files it reads
  (`selfcheck.cjs`, `trust.ts`, `trust.generated.cjs`) in `os.tmpdir()` -- never the tracked
  implementation files -- and asserts (1) an exact copy reports exit 0 / "in sync", (2) a drifted
  copy reports exit 1 + a stderr message matching `/DRIFT/`, (3) a missing committed file also
  reports exit 1 + `/DRIFT/`, and (4) `--write` regenerates a drifted copy back into sync
  (idempotent round-trip). **Adversarial proof the new test is non-vacuous:** before finalizing,
  built a throwaway scratch copy of `selfcheck.cjs` (never the tracked file) with its drift
  comparison forced to `if (false)` (simulating a broken drift-detector) and ran it against a
  drifted `trust.generated.cjs` copy -- it incorrectly printed "in sync" and exited 0, confirming
  that `selfcheck.spec.ts`'s `expect(result.status).toBe(1)` / `expect(result.stderr).toMatch(/DRIFT/)`
  assertions WOULD fail and catch exactly this class of regression. Converts this requirement's
  drift-detection leg from manual-proof-only to automated regression coverage.

**Result:** 3/4 requirements (TRUST-01, TRUST-08, TRUST-06-structure) were already automated with
genuinely adversarial, non-vacuous tests; 1 additional leg (TRUST-06 live behavior) is correctly
`human_needed`/live-CI, not a local-Vitest gap. TRUST-04 had one real automated-coverage gap (the
CLI drift-exit-1 path) which is now filled with `selfcheck.spec.ts`.
