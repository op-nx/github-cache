<objective>
MVP user story: As an external repo maintainer, I want the consumer contract (package exports,
action inputs, env knobs) enumerated and guarded by a test that fails on unintended change, so
that a dogfood refactor in this repo can never silently alter what I depend on (DOCS-05).

This is the load-bearing automated proof of the phase. It is a TDD feature: RED first (the guard
fails against a deliberately-wrong enumerated surface), then GREEN (the guard matches the real,
settled surface). It mirrors the pinned-deps.spec.ts precedent (D-05): an intentional surface
change edits the explicit lists HERE, so the contract change shows up as an obvious diff in the
spec file itself -- preferred over toMatchSnapshot(), whose `-u` regen is easy to rubber-stamp.

Purpose: scope the guard to the consumer contract ONLY (D-04's three groups + the fixed body
cap), never internal helpers, so it is stable against internal churn (ROADMAP Risk 1).

Output: packages/github-cache/src/public-surface.spec.ts.
</objective>
