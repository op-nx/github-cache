<objective>
MVP user story: As an external repo maintainer, I want a trust/security section that states
exactly which triggers write, the CREEP posture, and the adopter prerequisites, plus a versioning
statement that defines what 'breaking' means, so that I can adopt safely and know when an upgrade
can break me (DOCS-03, GOV-03).

This plan writes the two model-rendering docs: docs/trust-and-security.md (DOCS-03/D-08) and
docs/versioning.md (GOV-03/D-12), plus a content guard. The trust doc renders the SETTLED model
from Phase 5's single sources -- it never restates a guess -- and honors the CREEP posture: the
in-code host gate is defense-in-depth, containment is the sync gate + default-branch protection.

Purpose: this is the highest-stakes doc in the phase (a wrong security claim is worse than none).
It is a rendering of already-settled truth, with hard prohibitions against guessing the GHES
floor, framing retention as poison-containment, or encouraging fork-PR write tokens.

Output: docs/trust-and-security.md, docs/versioning.md, and a docs-trust content guard.
</objective>
